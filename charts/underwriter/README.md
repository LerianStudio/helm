# underwriter Helm Chart

## Chart Contract

- Chart type: `single-service`
- Required secrets: None for default render.
- Dependency notes: Bundles the Bitnami **PostgreSQL** (`architecture: replication`) and **Valkey** subcharts unless external services are configured.
- Production overrides: Provide production database credentials through dependency Secret settings (or external secret material); override image tags, ingress, resources, and persistence.
- Source/license: Source is in `github.com/LerianStudio/helm`; license is Apache-2.0.

### Infra secrets (single-source)

PostgreSQL and Valkey passwords are **single-sourced from their subchart-generated Secrets** and read
by the app via discrete `secretKeyRef` env entries — they are never duplicated into the app Secret:

| Env var | Source Secret (internal) | Key |
| --- | --- | --- |
| `POSTGRES_PASSWORD` | `<release>-postgresql` | `password` |
| `POSTGRES_REPLICA_PASSWORD` | `<release>-postgresql` | `replication-password` |
| `REDIS_PASSWORD` | `<release>-valkey` | `valkey-password` |

- When a subchart's `auth.existingSecret` is set, the ref points at that Secret instead.
- For **external** infra (`postgresql.external=true` / `valkey.external=true`) without an `existingSecret`,
  set `underwriter.secrets.POSTGRES_PASSWORD` / `POSTGRES_REPLICA_PASSWORD` / `REDIS_PASSWORD`; the app then
  reads them from its own Secret. Empty values are not written to the app Secret.
- If `valkey.auth.enabled=false`, no `REDIS_PASSWORD` env entry is wired at all.

Operator-provided secrets (no subchart source — set these yourself when the corresponding feature is on):
`RABBITMQ_DEFAULT_PASS`, `MULTI_TENANT_REDIS_PASSWORD`, `MULTI_TENANT_SERVICE_API_KEY`, `REDIS_CA_CERT`,
`SERVER_TLS_CERT_FILE`, `SERVER_TLS_KEY_FILE`.

### Release name

Infra hosts and subchart Secret names are derived **collapse-aware** from the release name via Bitnami's
`common.names.dependency.fullname`. The primary hosts resolve to `<release>-postgresql-primary` /
`<release>-valkey-primary:6379` and the Secrets to `<release>-postgresql` / `<release>-valkey`, honoring
any `nameOverride`/`fullnameOverride` and Bitnami name collapse. Any release name works. For external
infra, set `underwriter.configmap.POSTGRES_HOST` / `REDIS_HOST` explicitly.

Helm chart for deploying the Lerian underwriter service.

## Install

```sh
helm dependency build ./charts/underwriter
helm upgrade --install underwriter ./charts/underwriter \
  --namespace underwriter --create-namespace \
  --values ./charts/underwriter/values-template.yaml
```

## Required Configuration

- PostgreSQL and Valkey passwords are auto-generated by their subcharts and consumed via `secretKeyRef`
  (see "Infra secrets" above). To pin them, set `postgresql.auth.*` / `valkey.auth.*` or supply
  `postgresql.auth.existingSecret` / `valkey.auth.existingSecret`.
- Review `underwriter.configmap` for service URLs, multi-tenant settings, and telemetry configuration.
- Prefer existing Kubernetes Secrets when production secret material is managed outside Helm.

## Dependencies

This chart depends on the Bitnami PostgreSQL (replication) and Valkey subcharts. Run `helm dependency build` before rendering or installing from a local checkout.
